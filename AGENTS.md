# GitHub Proof Analyzer Agent Instructions

- Maintain proof-first language throughout the product.
- Avoid unfair scoring labels such as "good developer" or "bad developer."
- Keep analysis transparent and explain methodology clearly.
- Do not commit secrets, API keys, `.env` files, tokens, or `node_modules`.
- Run lint, typecheck, and build before final delivery.
- Make small meaningful commits and push after each successful milestone when a remote is available.
- Update `DEVELOPMENT_LOG.md` after every milestone with files changed, implementation notes, commit hash, and next step.
